Before even starting to write AWS commands, you need to have an AWS CLI.

You can download it from here,

[Installing or updating to the latest version of the AWS CLI - AWS Command Line Interface](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)

Now that you have the AWS CLI, First verify whether it’s installed correctly

```bash
aws --version // gives the aws version installed
```

Example response: aws-cli/2.15.30 Python/3.11.8 Windows/10 exe/AMD64 prompt/off

If you are facing any issues, troubleshoot using the installation guide link for which is given above.

First step after the AWS CLI Installation is to login to your AWS account.

For this, you need to have the Access Key ID, Secret Access Key and the region name.

### Configuration

```bash
aws configure
```

This will prompt you to enter the following

**AWS Access Key ID \[None\]:** *<enter-your-access-key-id>*

**AWS Secret Access Key \[None\]:** *<enter-your-secret-access-key>*

**Default region name \[None\]:** *<enter-your-region-name>*

**Default output format \[None\]:** *<can-leave-it-as-blank-or-specify-the-format>*

If you want to know whether, you have successfully logged in you can run this command

```bash
aws sts get-caller-identity
```

This command returns details about the IAM user or role whose credentials are used to call the operation.

## IAM (Identity and Access Management)

Important concepts in **IAM** are **Users, Groups, Roles, Policies, MFA** and many others

1.  Account Alias | Creation and Deletion
    
    ```bash
    aws iam create-account-alias --account-alias <value>
    aws iam delete-account-alias --account-alias <value>
    ```
    
    First command Creates an alias and second commands Deletes an alias for your Amazon Web Services account.
    
2.  List of Users
    
    ```bash
    aws iam list-users
    ```
    
    Lists the Users and their respective details such as *(Path, UserName, UserId, Arn, CreatedDate, PasswordLastUsed)*
    
3.  Change Password
    
    ```bash
    aws iam change-password --old-password <value> --new-password <value>
    ```
    
4.  Details of an IAM User
    
    ```bash
    aws iam get-user --user-name <value>
    ```
    
    Retrieves information about the specified IAM user, including the user’s creation date, path, unique ID, and ARN, Tags.
    
    If you do not specify a username, IAM determines the username implicitly based on the Amazon Web Services access key ID used to sign the request to this operation.
